### 브랜치 전략

---

- **main(master)**: 서비스을 직접 배포하는 역할을 하는 브랜치
- **feature**: 각 기능 별 개발 브랜치
- **develop**: feature에서 개발된 내용이 저장되는 브랜치
- **release**: 배포를 하기 전 내용을 QA(품질 검사)하기 위한 브랜치
- **hotfix**: main 브랜치로 배포를 하고 나서 버그가 생겼을 때 빨리 고치기 위한 브랜치
